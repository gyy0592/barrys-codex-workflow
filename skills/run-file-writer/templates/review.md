# Review

This file records fresh review of one completed spec. It must not review multiple specs at once.

## Spec

Write the spec id and short name here.

## Reviewer Context

List exactly what the reviewer received. The reviewer should receive only this spec's requirements, files, evidence, latest git evidence, and active constraints.

## Checks

- Does the evidence prove the spec success standard?
- Did the spec obey control/constraint.md?
- Did the spec stay within control/goal.md and the current spec?
- Did the work avoid future-spec tasks before this spec passed?
- Are unrelated files excluded from the checkpoint commit?
- Are missing facts or deviations recorded?
- Is it safe to move to the next spec?

## Findings

List every issue found. Write "none" if there are no issues.

## Result

Write PASS or FAIL.

## Required Fixes Before Next Spec

List required fixes. Write "none" only if the result is PASS.
